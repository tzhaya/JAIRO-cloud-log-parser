# WEKO3ソースにおけるアクセス統計集計ロジック

## 1. 目的

本資料は、WEKO3がファイルダウンロード数およびアイテム詳細画面の閲覧数を、どのような処理で記録・集計しているかを整理したものです。

対象とする主な統計は次のとおりです。

- No. Of File Downloads
- Open-Access No. Of File Downloads
- Detail View

## 2. ファイルダウンロードイベントの記録

WEKO3では、ファイルがダウンロードされた際に、ファイルダウンロードイベントを生成します。

イベントには、主に次の情報が保存されます。

- ファイルID
- ファイル名
- ファイルサイズ
- 対象アイテム
- 登録インデックス
- 利用者情報
- サイトライセンス情報
- 管理者・登録者等の利用者区分
- ファイルに設定された `accessrole`

特に重要なのは、統計イベントの `accessrole` が、アクセス時の状況から再計算されるのではなく、対象ファイルのJSONに保存された値をそのまま使用している点です。

概念的には次の処理です。

```python
accessrole = file_json.get("accessrole", "")
```

したがって、公開日を過ぎて現在は誰でもダウンロード可能なファイルであっても、ファイルメタデータが `open_date` のままであれば、イベントにも `open_date` が記録されます。

## 3. No. Of File Downloads の集計

`No. Of File Downloads` は、指定期間に記録されたファイルダウンロードイベント全体を対象とします。

集計時には、主に次の単位でまとめられます。

- ファイル名
- 登録インデックス名

各行には、次の値が出力されます。

- 総ダウンロード数
- 非ログイン利用者
- ログイン利用者
- サイトライセンス利用
- 管理者
- 登録者

総ダウンロード数は、対象となるダウンロードイベント全体の件数です。

## 4. Open-Access No. Of File Downloads の集計

`Open-Access No. Of File Downloads` は、同じファイルダウンロード統計データを対象としつつ、次の条件を追加して集計します。

```text
accessrole = open_access
```

そのため、両者の関係は次のとおりです。

```text
No. Of File Downloads
└─ そのうち accessrole = open_access のもの
   └─ Open-Access No. Of File Downloads
```

したがって、両者は独立した件数ではなく、後者は前者の部分集合です。

## 5. 二重クリック等の除外

WEKO3の統計処理では、同一利用者による短時間の重複アクセスを抑制するため、一定時間内の重複をまとめる処理があります。

ファイルダウンロードでは、ファイルID、接続元、セッション等を組み合わせた一意キーを作成し、短時間の重複イベントを除外します。

このため、Webサーバの生ログ上のリクエスト件数と、WEKO3統計上のダウンロード数は一致しない場合があります。

## 6. 利用者区分の集計

統計には、非ログイン、ログイン、サイトライセンス、管理者、登録者等の列があります。

これらはすべてが相互排他的な区分とは限りません。

例として、ログイン利用者がサイトライセンス対象ネットワークからアクセスした場合、同一アクセスが複数の観点に該当する可能性があります。

したがって、各内訳列を単純加算して総ダウンロード数を算出してはいけません。

総数は、出力された `No. Of File Downloads` の総数列を使用します。

## 7. アイテム詳細画面の閲覧数

アイテム詳細画面が表示されると、レコード閲覧イベントが生成されます。

主に次の情報が記録されます。

- レコードID
- アイテムタイトル
- 所属インデックス
- PID
- 利用者情報
- サイトライセンス情報

一方、アイテム閲覧イベントには、ファイルの `accessrole` は記録されません。

したがって、アイテム詳細画面の閲覧数と、OAファイルのダウンロード数は別の統計です。

## 8. 重要な整理

WEKO3の統計は、次のように理解する必要があります。

| 統計 | 対象 |
|---|---|
| No. Of File Downloads | 記録された全ファイルダウンロード |
| Open-Access No. Of File Downloads | `accessrole = open_access` のファイルダウンロード |
| Detail View | アイテム詳細画面の閲覧 |

`Open-Access No. Of File Downloads` は、現在誰でも利用可能なファイルを動的に判定して数えているわけではありません。

ファイルに保存された `accessrole` の値を条件として集計しています。
