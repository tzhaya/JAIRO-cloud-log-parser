# Issue #6 実装計画：Usage and Config シートに目的別の統計値選択ガイドを追加する

対象：[Issue #6](https://github.com/tzhaya/JAIRO-cloud-log-parser/issues/6) ／ 優先度 High ／ 実施順序 2 番目

## ゴール

利用者が「Usage and Config」シートだけを見て、目的に応じてどのシート・列を使うべきか判断できる状態にする。Issue #2（指標の使い分け）と #4（単純加算禁止）の Excel 側の受け皿を兼ねる。

## 変更対象

- Excel「Usage and Config」シートのみ（Power Query の変更なし）

## 作業手順

1. 「Usage and Config」シートの既存の使い方説明の下に「目的別ガイド」として次の表を追加する（docs/04 の表をそのまま採用）：

   | 目的 | 使用するシート・列 | 注意点 |
   |---|---|---|
   | リポジトリ全体のダウンロード量 | `File Download` シートの No. Of Times Downloaded | ログイン利用等を含む総数 |
   | 匿名利用の傾向 | `File Download` シートの Non-Logged In User | 必ずしも学外利用者のみではない |
   | ログイン利用 | `File Download` シートの Logged In User | ログイン状態の利用 |
   | 明示的OA設定ファイル | `Filedownload (OA)` シート | `accessrole = open_access` のみ |
   | アイテム詳細閲覧 | `Detail View` シートの View Count / Non-logged-in User | ファイルダウンロードとは別統計 |
   | ダウンロードの独自再集計 | `File Download` シートの各行・各列 | 用途に応じて対象列を選択 |
   | 詳細閲覧の独自再集計 | `Detail View` シートの各行・各列 | ダウンロードとは別統計 |
   | 現在公開中の全ファイル | 本レポートだけでは把握不可 | `open_date` 等の追加判定が必要 |

   ※実ワークブックの行レベル出力シート名は `File Download` と `Detail View`（間にスペースあり。クエリ名とは異なる）、OA シートは `Filedownload (OA)` です。表内はこの実在シート名で記載すること。
   ※注意：セル B9 の `Path` テーブル（クエリの入力元）の位置を動かさないこと。表は既存コンテンツの**下**に追加する。行挿入で B9 がずれる操作は行わない。
2. 表の直下に 2 つの補足を記載する：
   - 指標の選択は機関の運用・目的で判断すること、どちらか一方が正解ではないこと（Issue #2 の Excel 側対応）
   - 内訳列は相互排他でないため合計しないこと、総数は `No. Of Times Downloaded` を使うこと（Issue #4 の Excel 側対応）
3. ワークブックを日付サフィックス付きの新ファイル名で保存する（#3・#5 の Excel 変更とまとめてよい）。
