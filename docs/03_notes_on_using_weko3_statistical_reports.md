# WEKO3統計レポート利用時の注意点

## 1. この資料の目的

WEKO3の運用レポートを利用して、ファイルダウンロード数やアイテム閲覧数を集計する際の注意点を整理します。

## 2. レポート内の列を単純加算しない

ファイルダウンロードレポートには、次のような列があります。

- No. Of File Downloads
- Non-Logged In User
- Logged In User
- Site License
- Admin
- Registrar

これらの列は、すべてが互いに重複しない分類ではありません。

たとえば、ログイン利用者がサイトライセンス対象ネットワークからアクセスした場合、同じダウンロードが複数の観点に該当する可能性があります。

したがって、次のような計算は適切ではありません。

```text
Non-Logged In User
+ Logged In User
+ Site License
+ Admin
+ Registrar
```

総ダウンロード数を確認する場合は、`No. Of File Downloads` の総数列を使用します。

## 3. No. Of File Downloads と Open-Access No. Of File Downloads

### No. Of File Downloads

指定期間中に記録されたファイルダウンロード全体です。

### Open-Access No. Of File Downloads

全ダウンロードのうち、ファイルに次の設定が保存されているものだけを集計します。

```text
accessrole = open_access
```

したがって、両者は次の関係です。

```text
No. Of File Downloads
└─ Open-Access No. Of File Downloads
```

両者を加算すると二重計上になります。

## 4. OA統計は「現在公開中の全ファイル」ではない

`Open-Access No. Of File Downloads` という名称から、現在誰でも利用可能なファイルのダウンロードをすべて集計しているように見えます。

しかし、実際の条件はファイル設定値が `open_access` であることです。

そのため、次のようなファイルは、現在誰でもダウンロード可能でもOA統計に含まれません。

- 公開日を過ぎた `open_date`
- アイテムのアクセス権は open access だが、ファイルの `accessrole` が異なるもの
- 旧システム移行時に異なる設定値が入ったもの
- `accessrole` が空欄・未設定のもの

## 5. 非ログイン利用者の意味

`Non-Logged In User` は、WEKO3にログインしていない状態で行われたダウンロードを示します。

この値は、管理者・登録者等のログイン利用を除外して、一般利用の傾向を把握する目的には有用です。

ただし、次の点に注意が必要です。

- 非ログイン利用者が必ず学外者とは限らない
- 学内利用者がログアウト状態で利用する場合もある
- 担当者が未ログインで確認する場合もある
- サイトライセンス対象ネットワークから匿名利用する場合もある

したがって、`Non-Logged In User` は次のように表現するのが適切です。

> 非ログイン状態で行われたダウンロード数

または、

> 匿名利用によるダウンロード数

「学外利用者数」や「外部利用者数」と断定するのは避けるべきです。

## 6. 利用目的ごとの推奨値

| 利用目的 | 使用する値 |
|---|---|
| 全体のダウンロード数 | No. Of File Downloads |
| 匿名利用の把握 | Non-Logged In User |
| ログイン利用の把握 | Logged In User |
| 明示的にOA設定されたファイルの利用 | Open-Access No. Of File Downloads |
| 現在誰でも利用可能なファイルの全利用 | 運用レポートだけでは正確に把握できない |

## 7. ファイル名とインデックス名の扱い

WEKO3のレポートは、同じファイル名でも登録インデックスが異なる場合、別行として出力されることがあります。

複数月のレポートを結合する際は、少なくとも次の単位を維持する必要があります。

- 年月
- ファイル名
- 登録インデックス名

ファイル名だけで集約すると、異なるインデックスの値が混在する可能性があります。

## 8. 月別集計時の注意

Power Query等で複数月を結合する場合は、元レポートに年月列を追加してから結合する方法が適切です。

ピボットテーブルでは、集計目的に応じて値フィールドを選択します。

例：

- 総利用を見る場合：No. Of File Downloads
- 匿名利用を見る場合：Non-Logged In User
- OA設定ファイルを見る場合：OAセクションの No. Of File Downloads

## 9. 差異が生じた場合の確認手順

全ダウンロードとOAダウンロードに差がある場合は、次の順で確認します。

1. 同じ年月、ファイル名、登録インデックス名で比較する
2. OAセクションに該当行があるか確認する
3. アイテムJSONの `item_files[].accessrole` を確認する
4. `open_date` の場合は公開日を確認する
5. アイテムレベルのアクセス権とファイルレベルの `accessrole` を区別する
6. 旧データ移行による設定差がないか確認する

## 10. 推奨する説明表現

### No. Of File Downloads

> WEKO3で記録されたファイルダウンロード全体

### Non-Logged In User

> WEKO3にログインしていない状態で行われたダウンロード

### Open-Access No. Of File Downloads

> ファイルの `accessrole` が `open_access` に設定されているダウンロード

「現在誰でも利用できるファイルのダウンロード数」という説明は、`open_date` 等が除外されるため正確ではありません。
